Visualização do Serviço

 Top  Previous  Next

Para o usuário final com a permissão de visualizar instância de serviço no controle de acesso da instância, é disponibilizada uma lista relatórios cadastrados, para que consultas de dados estatísticos do portal possam ser extraídas.

Por default, ao selecionar um relatório são fornecidos os campos Número de itens e datas inicial e final da consulta. O número de itens refere-se a quantidade de registros a serem exibidos, independente se o resultado da consulta possuir mais registros que o solicitado nesse campo. Por default, esse valor é 10 itens. A partir de determinado valor, é gerada uma paginação. O resultado da consulta é exibido em um IFRAME após acionar visualizar. Para salvar o relatório, basta acionar uma das imagens acima do resultado da consulta. Ele pode ser salvo nos padrões de arquivo do Excel, Word e PDF .

Será exemplificado a visualização para cada relatório cadastrado, correspondente a um dos tipos de relatório: pageviews, hourlyvisitors, dailyvisitors, pagegeneration, interfacegeneration,xmlgeneration, ratioxmlhtml, hourlypagecacheclear, pagecacheclear.

A figura abaixo exemplifica uma consulta do relatório "Páginas visitadas", do tipo pageviews criado para o canal "Intranet Padrão".

Ao selecionar este relatório na lista, além dos campos de preenchimento padrão são exibidos também a hora das datas inicial e final para realizar a consulta. Caso não sejam preenchidos, serão adotados 30 dias da data atual para a data inicial e, a data atual para a data final. Já quando a data é preenchida e a hora não, é assumido zero hora daquela data.

O resultado exibe o título do relatório (atribuído no cadastro do relatório) e uma tabela com a lista de páginas de acordo com a agregação escolhida pelo administrador do serviço, durante o cadastro de relatórios. No caso, a agregação é o próprio nome da página. Exibe ainda a quantidade de visualização das páginas do canal e o tempo de máximo, médio e mínimo de processamento. A ordenação está por ordem decrescente de quantidade de visualização de páginas.

 

Observe que o período informado nos parâmetros não é o mesmo exibido no resultado. Isso porque o resultado exibe o período exato que o conjunto de páginas daquele canal foram acessadas, dentro do período especificado pelo usuário.

Observação

A figura abaixo exemplifica uma consulta do relatório "Limpezas de cache por hora", do tipo hourlypagecacheclear criado para o canal "Portal".


 

Ao selecionar este relatório na lista, são exibidos os seguintes campos para preenchimento:

•Data inicial: Campo opcional. Indica a data inicial da pesquisa dos dados. Caso omitido é considerado a data atual.
•Data final: Campo opcional. Indica a data final da pesquisa dos dados. Caso omitido é considerado a data atual.
•Filtro de login, Filtro de caminho de canal, Filtro de identificador de servidor, Filtro de identificador de serviço, Filtro de grupo do usuário: Campos opcionais. Indicam qual o tipo de filtro que será aplicado ao campo a ser filtrado. Pode ser um dos valores: “Contém”, “Igual”, “Começa com” ou “Termina com”.
•Login do usuário: Campo opcional. Caso fornecido filtra os resultados pelo login do usuário dos eventos de limpeza de cache com o login informado, utilizando o tipo de filtro indicado.
•Caminho do canal: Campo opcional. Caso fornecido filtra os resultados pelo caminho do canal dos eventos de limpeza de cache com o caminho do canal informado, utilizando o tipo de filtro indicado. É importante salientar que o filtro por caminhos de canal cujos nomes são traduzidos através de strings no portal (por exemplo, “STR_MY_WEBSITE_CHANNEL”) não são armazenados com suas traduções, mas sim com as strings originais. Ou seja, caso exista um canal com nome “STR_MY_WEBSITE_CHANNEL” e exista uma string cadastrada com chave “STR_MY_WEBSITE_CHANNEL” e valor “Meu website” para Português, os eventos de limpeza de cache ocorridos dentro desse canal terão seu caminho armazenado como “.../STR_MY_WEBSITE_CHANNEL/...” e não “.../Meu website/...”. Nesse caso, caso seja pesquisado por caminho de canal contendo “Meu website” nenhum resultado seria retornado.
•Identificador de servidor: Campo opcional. Caso fornecido filtra pelo identificador do servidor no qual o evento de limpeza foi disparado com o valor dado, utilizando o tipo de filtro fornecido.
•Identificador de serviço: Campo opcional. Caso fornecido filtra pelo identificador do serviço no qual o evento de limpeza foi disparado com o valor dado, utilizando o tipo de filtro fornecido.
•Grupo do usuário: Campo opcional. Caso fornecido filtra pelos grupos dos quais o usuário que disparou o evento de limpeza é membro com o valor dado, utilizando o tipo de filtro fornecido.

O resultado exibe o título do relatório (atribuído no cadastro do relatório) e uma tabela com a lista de total de limpezas de cache por hora, dentro do período especificado para a consulta.

Esse relatório fornece os picos de limpezas de cache por hora.

Observação: É importante salientar que o horário das ocorrências é relativo à limpeza do cache, não à geração do mesmo.
