•ID: Guid atribuído automaticamente pelo portal após confirmar a criação da página. Essa identificação, portanto é visualizada somente na edição das propriedades da página. Pode ser utilizado para verificações de problemas no banco de dados, por exemplo;

 

•Gerar identificador amigável automaticamente: Indica se o identificador amigável deve ser gerado automaticamente pelo portal.

 

•Identificação amigável: Identificador amigável que representa essa página. Esse identificador não pode se repetir em duas ou mais páginas que pertençam ao mesmo canal. Quando gerado automaticamente, o identificador amigável utiliza o nome da página como base para a geração, com algumas transformações realizadas para garantir a unicidade entre elementos filhos do mesmo pai. Quando não gerado automaticamente, esse campo é obrigatório e para seu preenchimento recomenda-se somente o uso de caracteres alfa-numéricos minúsculos não acentuados como valor desse campo. Esse campo tem um limite de 50 caracteres.

 

Observação

 

Quando uma página tem seu identificador amigável gerado automaticamente e tem seu nome alterado ou é movida seu identificador amigável será regerado.

 

•Server Side Include: SSI é uma tecnologia utilizada com páginas HTML. Algumas tags específicas no HTML são tratadas e substituídas no servidor, para dinamicamente incluir conteúdo que não faz parte do HTML original.

Essa tecnologia é uma alternativa, portanto, para inserir porções de informações que são atualizadas constantemente, sem que as  páginas precisem ser geradas novamente para receber e exibir tais informações.

A decisão de quando utilizar SSI, e de quando ter que gerar novamente a página, vem da necessidade de analisar se a página deve ser estática ou não, e se deve ser gerada a cada atualização. Verificar o cenário de SSI em cache de página com essa tecnologia.
 

□Sobrescrever padrão: Habilitando esta propriedade, será possível alterar a configuração doServer side include para a página;
 
□Habilitado: Por default, a configuração de server side include (SSI) é calculada automaticamente. Isso significa que automaticamente é habilitada para "sim" quando alguma interface da página estiver com o SSI habilitada. Dessa forma, quando a página for cacheada será gerado um HTML com extensão SHTML.

 

•Protocolo: Permite determinar qual protocolo será utilizado durante a navegação no canal e suas páginas. O Lumis fornece quatro opções:
□Herdar configuração: herda o que está definido para o canal pai (este valor default para a opção protocolo);
□Somente http: força o uso do protocolo http na página, redirecionando para a mesma página com o protocolo http;
□Somente https: força o uso do protocolo https para a página, redirecionando para a mesma página com o protocolo https;
 

Observação

 

A utilização desse protocolo desabilitará o cache HTML da página. Isso será visível no Gerenciador de Cache de Páginas. Porém, o valor para a propriedade Gerar Arquivo HTML das propriedades da Página não será alterado.
□Permitir todos: a página será exibida com qualquer protocolo válido digitado na url.

 

Observação

 

É necessário verificar se o arquivo de configuração do servidor de aplicação contém as configurações necessárias para utilização do HTTPS.

Verificar também a necessidade de configuração no arquivo lumisportalconfig.xml da porta a ser utilizada.

 

•URLs Amigáveis: O funcionamento de urls amigáveis para página é similar ao de canal (Verificar primeiramente a configuração de url amigável de canal). Também é necessário habilitar para "sim" quando não estiver herdando essa configuração do canal "pai". A diferença está na propriedade Sobrescrever o formato.
□Habilitar: Permite determinar se será habilitado a propriedade de url amigável para a página, caso a herança não esteja habilitada (e canal com a propriedade habilitada para sim). Permite também, que em uma estrutura de canais "pai" com url amigável configurada, se desejar que determinada página não possua url amigável, basta configurar para não. Dessa forma, será exibido o caminho dinâmico ou estático (se a página estiver em cache). Por fim, pode simplesmente herdar o que está configurado para o canal pai;
□Formato: É possível, através dessa propriedade, determinar outro formato para a url amigável, caso a página esteja herdando a url amigável do canal pai.

•Excluir Web Resources: Excluir Web Resources significa excluir todos os Web Resources relacionados a página.

Observações

Ao excluir um Web Resource, a URL amigável que correspondia a ele deixa de estar disponível. Com isso links para ele, seja externo ao portal ou um link interno que esteja em cache, estarão referenciando uma página que não existe.
