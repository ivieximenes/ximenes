O serviço pode ser montado de diversas maneiras, para a navegação do usuário da solução final, dependendo de como foi arquitetado pelo administrador do portal.

Para utilizar o serviço Barra de Navegação, basta instanciá-lo em um canal a partir do qual deseja-se que haja uma navegação, e arrastar suas interfaces para as áreas das páginas em modo de edição. Suas interfaces correspondem a um ambiente administrativo de itens de menu e interfaces de interação com o serviço e portal pelo usuário final.

•Administração: Esta interface permite o cadastro e manutenções (alterações e exclusões) dos itens de menu;
 
•Caminho da página: Esta interface exibe a hierarquia de canais até a página corrente.

Os itens de menu do “caminho da página” aparecem no sentido horizontal, um ao lado do outro.

Exemplo:

No caso do serviço estar instanciado no canal “Notícias”, que possui o canal “Esportes”, e este possui a página “Esportes Radicais”, quando o visitante estiver nesta página, a interface Caminho da Página mostrará a seguinte hierarquia na página pertencente ao canal “Esportes Radicais”:

Notícias>Esportes>Esportes Radicais

Quando o item de menu selecionado do caminho da página for um canal, será exibida a primeira página deste canal.

Para as páginas que contiverem interface de detalhes, será exibido o título do conteúdo da interface de detalhes no lugar do nome da página, desde que esta página esteja configurada como oculta. Caso contrário, será exibido o nome da página.

 

•Menu: Interface que permite exibir a barra de navegação para o usuário final da solução .

Geralmente, todas as páginas de um portal possuem recursos de menu e caminho da página. Uma prática comum para mostrá-los em todas as páginas, é aplicá-los em um template, que por sua vez é utilizado pelas páginas do portal.

Observações

 
1.Ao instanciar a interface Administração em uma página, por default é exibido um item de menu cadastrado que contém a estrutura hierárquica de canais e páginas, a partir do canal que possui a instância de serviço Barra de Navegação. A interface Menu exibe esta estrutura na forma de um menu vertical.
2.Caso a barra de navegação represente a estrutura do portal, pode-se determinar quais delas serão exibidas no menu, através da propriedade Oculto de cada página ou canal. Templates de página e canal não são referenciados na estrutura de menu.
3.Caso haja necessidade de ordenação das páginas e canais dentro do menu (o padrão é a ordenação alfabética), determinar valores para cada uma delas no campo Posição, nas propriedades das páginas e canais. O menu vai seguir a ordem numérica de posições no sentido ascendente.
4.Quando o item é uma página, o link leva diretamente para ela. Quando o item é um canal, o link leva diretamente para a primeira página deste canal. Caso o canal não possua páginas, será exibida a primeira página do canal filho, e assim por diante. Caso contrário, será remetido para a página principal padrão ou página principal do portal;
5.Acesso a canais e páginas do menu, sem permissão, será exibido a página de login.

 

O ambiente administrativo de barra de navegação requer apenas uma página em um canal restrito do portal, onde apenas os administradores terão acesso, após o login. A fim de proporcionar uma maior organização no portal, é aconselhável que esta página pertença a um canal específico, o qual está sendo chamado aqui de “Barra de Navegação”.


 

Observação

 

Para informações sobre como instanciar canais e páginas, consultar a seção sobre Portal Studio .

Alterações estruturais no navegador (adição/exclusão de canais e páginas) não atualizarão o cache das páginas que possuem a interface Menu do serviço Barra de Navegação. Isso ocorre porque um serviço que utiliza funcionalidades mas que não administra dados, não há como saber quando a informação a informação foi alterada;
