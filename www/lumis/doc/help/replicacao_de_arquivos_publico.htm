Replicação de Arquivos Públicos

 Top  Previous  Next

 
O Lumis Portal oferece um serviço para replicação de arquivos públicos. Nesse serviço, é possível manter um cadastro de replicações.  O observador WebFileReplicationObserver captura os eventos no sistema de arquivos do cluster (novo arquivo, alteração de arquivo e deleção) e chama cada uma das replicações cadastradas para replicar o evento.

O serviço pode ser encontrado em Deployment e Performance Tuning -> Replicação de Arquivos Públicos.

 

Serviço de Replicação de Arquivos Públicos:

 

Com o serviço de replicação de arquivos públicos é possível realizar as seguintes operações:

•Adicionar uma replicação
•Editar uma replicação
•Remover uma replicação

 

Adicionar Replicação

Para adicionar uma página, basta selecionar a opção “Adicionar”.

Tela Adicionar Replicação de Arquivos Públicos:


 

As informações necessárias para cadastrar uma replicação são:
Campos:

•Nome
Nome para a replicação. Campo livre e obrigatório.
•Website

 Website o qual será utilizado para determinar o contexto da replicação. Campo obrigatório.

•Tipo
Tipo de replicação: FTP ou sistema de arquivos local. Campo obrigatório.
•Filtro Canal
Caso seja preenchido, apenas os arquivos que correspondem a cache de página dentro da estrutura do canal escolhido ou que estejam associados a instâncias de serviço dentro da estrutura do canal escolhido terão seus eventos replicados. Só serão exibidos canais a partir do canal raiz do website selecionado. Porém, não serão omitidos subcanais pertencentes a outros websites internos ao website raiz. Campo opcional.
•Filtro Caminho
Caso seja preenchido, apenas os arquivos que possuem caminho relativo correspondendo ao padrão especificado no filtro terão seus eventos replicados. Campo opcional.

De acordo com o tipo de replicação selecionado, é necessário preencher propriedades adicionais.

Replicação do tipo FTP:


 

Como podemos ver, para replicações do tipo FTP as propriedades adicionais são:

•Host
Host FTP.
•Password
Password do host FTP.
•Port
Porta do host FTP.
•Username

Username do host FTP.
 

 

Replicação do tipo sistema de arquivo local:

Na replicação do tipo local a propriedade “Path” deve ser informada com o diretório onde os arquivos serão salvos.

 

Editar Replicação

Para editar uma replicação é necessário selecionar o item e selecionar a opção “Editar”.
Para referencias aos campos veja descrição acima.

 

Excluir Replicação de Arquivos Públicos

Para excluir uma replicação, é necessário selecionar o item a ser excluído e selecionar a opção “Excluir”. Nesta opção, é possível excluir mais de uma replicação por vez, selecionando os itens que se deseja excluir. O sistema exibe uma mensagem de confirmação:

Basta confirmar e a replicação será excluída. Com a exclusão, novos eventos não serão replicados para esta configuração.
