O Lumis Portal é  um ambiente para criação de websites e soluções de Portais. Possui funcionalidades determinantes a essas soluções, tais como: controle de acesso, personalização, single-sign on, importação de usuários e grupos, integração de aplicações, caching, agregação de conteúdo, busca, multilingüe, serviços de gerenciamento de conteúdos e serviços de colaboração seguindo a necessidade do mercado por aplicações web 2.0.

Todas essas funcionalidades estão integradas e estão acessíveis a partir de uma camada de apresentação rica, flexível, eficiente e de fácil uso.

É uma aplicação JavaTM J2EE 1.4, podendo ser executada em um servidor de aplicação compatível com Servlet 2.4 e JSP 2.0. Para o seu total funcionamento também é necessário um servidor de banco de dados compatível.

A arquitetura do Lumis Portal dá suporte a portlets JSR-168, que é um padrão adotado na criação de portais para web .

Para suportar as soluções web, o ambiente permite a criação de uma estrutura hierárquica baseada em canais onde são organizadas as páginas. Ele permite integrar em uma mesma página interfaces de diversas aplicações diferentes. Aqui, as aplicações são chamadas de serviços e, suas telas, de interfaces.

Os serviços são componentes  que estendem as funcionalidades do portal. Um serviço pode ser uma simples aplicação que mostra uma imagem, um fórum de discussão ou mesmo uma aplicação complexa como o CRM da empresa. Esses serviços podem estar fisicamente no mesmo servidor do portal ou podem estar em outros servidores.

O Lumis Portal vem com diversos serviços prontos para uso, e permite a implementação de novos serviços, tanto declarativamente, por XMLs de configuração, quanto programaticamente, através da extensão das classes Java do produto.

A camada de apresentação dos serviços é dividida em interfaces. Estas são plugáveis nas páginas do Portal e podem ser de três tipos básicos: formulário, lista e detalhe. Dependendo do serviço, ele pode ter tantas interfaces quantas forem necessárias.

As interfaces são customizáveis por XSL, o que significa que elas utilizam os dados a serem apresentados para montar um XML de informações, que, posteriormente, é transformado por XSL para geração do HTML que será enviado ao browser. Isto permite que as interfaces sejam reutilizadas em vários contextos, apenas especificando-se o XSL utilizado em cada caso.

O Lumis Portal permite que sejam criadas soluções acessíveis, de acordo com o padrão do W3C. Todos os códigos e layout de páginas geradas automaticamente pelo Lumis Portal, e que não podem ser modificados pelos usuários, baseiam-se nas recomendações para acessibilidade contidas nos documentos “Web Content Accessibility Guidelines 1.0 ” e “Checklist of Checkpoints for Web Content Accessibility Guidelines 1.0 ”, fornecidos pelo W3C.
