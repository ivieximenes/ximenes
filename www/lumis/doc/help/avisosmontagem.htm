Montagem

O serviço pode ser montado de diversas maneiras, para o editor de conteúdo e para o usuário final da solução, dependendo de como foi arquitetado pelo administrador do portal.

Para utilizar o serviço de Avisos, basta instanciá-lo em um canal e arrastar suas interfaces para as áreas das páginas em modo de edição. Suas interfaces correspondem a um ambiente administrativo de cadastro do conteúdo e áreas de visualização do serviço pelo usuário final.

 

•Administração: Esta interface permite o cadastro e manutenção do conteúdo de avisos pelo editor de conteúdo;
•Lista: Esta interface disponibiliza a lista de avisos cadastrados, para o usuário final da solução. Somente estarão disponíveis nesta lista aqueles avisos que estão dentro do período de publicação solicitado no cadastro;
•Lista Rápida: Esta interface exibe uma lista de assuntos cadastrados mais recentemente, de acordo com o período de publicação, gerando um link "mais" para uma lista completa deles. A quantidade de assuntos na Lista Rápida pode ser determinada através da opção “Número de itens” disponível no right-click desta interface;
•Detalhes: Esta interface exibe os detalhes do aviso quando este é selecionado nas interfaces Lista e Lista Rápida.
 

Para disponibilizar o serviço de Avisos para o usuário final, basta instanciar as interfaces Lista, Lista Rápida e Detalhes em páginas diferentes do canal. A página que contém a interface Detalhes deve ter a propriedade “oculto” selecionada, pois esta página será acessível a partir das páginas correspondentes às interfaces Administração, Lista, Lista Rápida, e até mesmo Busca com Resultados (do serviço de Busca ).
 

O ambiente de administração de avisos requer apenas uma página em um canal restrito do portal, onde apenas os editores de conteúdo terão acesso, após o login. A fim de proporcionar uma maior organização no portal, é aconselhável que esta página pertença a um canal específico, o qual está sendo chamado aqui de “Avisos”.

 

 

Atribuição de Permissões

Para que o editor de conteúdo possa acessar o ambiente administrativo de avisos, o administrador do portal deve atribuir a ele a permissão “Editar conteúdo de instância de serviço”. Esta permissão é concedida no controle de acesso da instância do serviço.

Na figura abaixo, há um exemplo do controle de acesso de uma instância de serviço, cujo usuário “Carla” está com permissão “Editar Conteúdo”, atribuída pelo administrador do portal.

Para que o editor de conteúdo realize upload de arquivos, é necessário que ele tenha a permissão de Efetuar Upload de arquivo no Gerenciador de acesso do portal.

 

Observação

 

O Lumis Portal oferece como sugestão de montagem deste serviço um prebuilt channel com uma estrutura de canal, páginas, e disposição das interfaces nas páginas.

Para informações sobre como instanciar canais, páginas, e atribuição de permissões no controle de acesso, consultar a seção sobre Portal Studio .

Através do right-click da interface, encontram-se disponíveis parametrizações para a instância de serviço e customizações para a interface. Consultar a seção Customização de instâncias de Interfaces. Entre elas o repositório de imagens a ser utilizado no editor HTML.
