•ID: Guid atribuído automaticamente pelo portal após confirmar a criação do canal. Essa identificação, portanto é visualizada somente na edição das propriedades do canal. Pode ser utilizado para verificações de problemas no banco de dados, por exemplo;

 

•Gerar identificador amigável automaticamente: Indica se o identificador amigável deve ser gerado automaticamente pelo portal.

 

•Identificação amigável: Identificador amigável que representa esse canal. Esse identificador não pode se repetir em dois ou mais canais que pertençam ao mesmo canal pai. Quando gerado automaticamente, o identificador amigável utiliza o nome do canal como base para a geração, com algumas transformações realizadas para garantir a unicidade entre elementos filhos do mesmo pai. Quando não gerado automaticamente, esse campo é obrigatório e para seu preenchimento recomenda-se somente o uso de caracteres alfa-numéricos minúsculos não acentuados como valor desse campo. Esse campo tem um limite de 50 caracteres.

 

Observação

 

Quando um canal tem seu identificador amigável gerado automaticamente e tem seu nome alterado ou é movido seu identificador amigável será regerado.

 

•Protocolo: Permite determinar que protocolo será utilizado durante a navegação no canal e suas páginas. O Lumis fornece quatro opções:
□Herdar configuração: herda o que está definido para o canal pai (valor default);
□Somente http: força o uso do protocolo http no canal e suas páginas (caso essas estejam herdando esta configuração de protocolo), redirecionando para a mesma página com o protocolo http;
□Somente https: força o uso do protocolo https para o canal e suas páginas (caso essas estejam herdando esta configuração de protocolo), redirecionando para a mesma página com o protocolo https;
 

Observação

 

A utilização desse protocolo desabilitará o cache HTML das páginas do canal. Isso será visível no Gerenciador de Cache de Páginas. Porém, o valor para a propriedade Gerar Arquivo HTML das propriedades do Canal não será alterado.
□Permitir todos: o canal e suas páginas (caso essas estejam herdando esta configuração de protocolo) são exibidos com qualquer protocolo válido digitado na url.

 

Observação

 

É necessário verificar se o arquivo de configuração do servidor web contém as configurações necessárias para utilização do HTTPS.

 

•Habilitar Grupos Locais: Ao selecionar esta propriedade, opções para cadastro de grupos e usuários locais são disponibilizadas na área administrativa do canal:

Usuário com permissão de gerência de grupos locais (no controle de acesso do canal) pode adicionar, editar, excluir e incluir grupos e usuários locais somente.

A vantagem da utilização de grupos locais está na exportação e importação de canais, e na aplicação de templates, por exemplo.

 

•Prefixo: Este campo estará disponível para edição obrigatória, quando a opção “Habilitar Grupos Locais” estiver selecionada. Sendo assim, todos os usuários e grupos locais criados terão o prefixo especificado antes do login, separados por um “.”;
 
•Tipo: Permite atribuir um nome à identificação do canal no registro de log, por exemplo, facilitando sua localização. Outra utilidade, é para o serviço de busca. O tipo pode ser: definido pelo usuário; Usar o nome do canal; ou o caminho do canal “pai”;
 
•Caminho: Caso fique a critério do usuário definir este caminho, o caminho amigável terá um nome fictício;

 

•URLs Amigáveis: As urls das páginas de um portal, quando em uma forma de fácil identificação, são mais "legíveis" para um usuário e de fácil indexação para um mecanismo de busca. É mais interessante que a URL traga o assunto da página, e não somente o título e a descrição da mesma, para que um mecanismo de busca como Google, por exemplo, identifique que determinada página trata do assunto pesquisado pelo usuário. Portanto,configurando a propriedade URLs Amigáveis, ao invés da url da página conter guids e parâmetros, passará exibir paths legíveis que podem refletir a localização da página bem como termos de seu conteúdo.

 

Além disso, todas as urls no escopo aonde essa propriedade foi definida, possuirão a extensão default .htm independente da página ser dinâmica ou estática (Cache de página).

 

A extensão do arquivo pode ser configurada no arquivo de configuração do portal lumisportalconfig.xml, mais especificamente onde é configurad a extensão do cache html.

 

Basicamente, configura-se da seguinte forma:
 

□Habilitar: Permite habilitar a propriedade url amigável para o canal e seu escopo. O valordefault é herdar configuração do canal pai. Para habilitar a propriedade, basta escolher a opção "sim". Útil quando este canal estiver herdando uma configuração de um canal para qual a url amigável não esteja configurada. Pode-se também determinar o valor "não", quando por exemplo, em uma estrutura hierárquica de canais, deseja-se determinar aonde interrompe o escopo do Formato da url amigável;
 
□Formato Herdável: permite especificar o formato da url do canal e dos filhos (canais e páginas) caso a herança neles esteja habilitada.
 
Este formato padrão para quando a propriedade está habilitada é ${ancestralPath}/${wr.title}${paramsPath}, que é uma property bag lumis.portal.url.IURLResolver.pattern cujo valor é configurado através de Expression Language composta pelas seguintes variáveis:

 

- ${ancestralPath}: Corresponde ao caminho amigável do canal pai (ou web resource) mais próximo que tenha a propriedade url amigável configurada;

 

- ${wr.title}: Corresponde a propriedade título do web resource (canal ou página, por exemplo). Geralmente, o último title é o da página;
 

- ${paramsPath}: Juntamente com ${wr.title} adiciona uma parametrização caso o ${wr.title} do web resource irmão, seja repetido.
 

□Formato específico: permite especificar um formato diferente para o canal corrente enquanto os filhos adotarão o especificado no campo acima, ou herdar o formato do canal pai. Portanto, o Formato Específico não é herdado.

 

Observações

 

Tanto para o campo Formato quanto para o campo Formato Específico, o valor deve começar com "/";

Consultar o javadoc GenericURLResolver para maiores detalhes sobre as Expression Language;

A url amigável sofre os seguintes tratamentos:

•Conversão de maiúsculo para minúsculo;
•Remoção de acentuação;
•Substituição de caracteres especiais para "-";
•Redução de mútiplos "-" para um único e remoção de "-" do final;

Consultar javadoc lumis.portal.url para customizar url amigável através de property bag.

Como todos os canais e páginas abaixo do canal "Esportes", inclusive ele, terão a mesma url amigável, o portal acrescenta um parâmetro numérico(crescente a cada acesso) para diferenciar uma url de cada web resource, evitando assim um conflito.

Observações

 

1. Quando o formato de uma url amigável é alterado, é necessário remover os registros gerados no Gerenciador de Web Resource, pois o portal continuará servindo as urls antigas. Porém, a remoção de web resource para regerar as urls, gera impacto por deixar os caminhos correspondentes inválidos;

 

2. Como boa prática, os caminhos das urls amigáveis devem ser arquitetados de forma organizada, evitando coincidir com caminhos de outros recursos existentes na aplicação;

 

3. O caminho amigável de cada web resource deve ser único. Por isso um portal com diversas áreas deve ser organizado com políticas organizacionais para evitar conflitos de caminhos.

 
 

•Excluir Web Resources: Excluir Web Resources significa excluir todos os Web Resources relacionados ao canal e seus descendentes (páginas e canais).

Observações

Ao excluir um Web Resource, a URL amigável que correspondia a ele deixa de estar disponível. Com isso links para ele, seja externo ao portal ou um link interno que esteja em cache, estarão referenciando uma página que não existe.
