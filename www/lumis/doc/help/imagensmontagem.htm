O serviço pode ser montado de diversas maneiras, para os editores de conteúdo e para o usuário final da solução, dependendo de como foi arquitetado pelo administrador do portal.

Para utilizar o serviço de Repositório de mídias basta instanciá-lo em um canal e arrastar suas interfaces para as áreas das páginas em modo de edição. Suas interfaces correspondem a um ambiente administrativo de cadastro do conteúdo e áreas de visualização do serviço pelo usuário final.

 

•Administração: Esta interface permite a adição e manutenção de mídias e pastas pelo editor de conteúdo no repositório. Os arquivos são criados fisicamente em lumisdata/data/files ;
•Lista: Esta interface disponibiliza a lista de mídias para o usuário final, adicionadas ao repositório para download;
•Detalhe: Esta interface permite exibir informações detalhadas sobre o arquivo de imagem selecionado na página que contém a interface Lista, pelo usuário da solução;

Para disponibilizar o serviço de Imagens para o usuário final, basta instanciar as interfaces Lista e Detalhes em páginas diferentes do canal.

O ambiente de administração de arquivos de imagens requer apenas uma página criada em uma área restrita do portal, onde apenas os editores de conteúdo terão acesso.

A fim de proporcionar uma maior organização no portal, é aconselhável que esta página seja colocada dentro de um canal específico, o qual está sendo chamado aqui de “Imagens”.

 

Atribuição de Permissões

Para que o editor de conteúdo possa acessar o ambiente administrativo do repositório de mídias, o administrador do portal deve atribuir a ele a permissão “Editar conteúdo de instância de serviço”. Esta permissão é concedida no controle de acesso da instância do serviço.

Na figura abaixo, há um exemplo do controle de acesso de uma instância de serviço, cujo usuário Carla está com permissão “Editar Conteúdo”, atribuída pelo administrador do portal.

Para que o editor de conteúdo realize upload de arquivos, é necessário que ele tenha a permissão de Efetuar Upload de arquivo no Gerenciador de acesso do portal .

Parametrização da Instância de Serviço

A “Propriedades de Gerenciamento de Arquivos” acessada no right-click das instâncias de interfaces, juntamente com as outras customizações, permite determinar se as mídias do repositório serão públicos ou privados.

Caso os arquivos sejam públicos, as interfaces de outros serviços que realizam download de arquivos do repositório de imagens, terão acesso direto, sem autenticação do usuário (os arquivos estarão localizados em “www/data/files”). Caso contrário, se os arquivos forem privados, estarão em uma área restrita (acessados via jsp), havendo necessidade de autenticação pelo portal.

Uma vez que o repositório seja configurado como público, os aquivos já existentes bem como os que forem adicionados após tal configuração, será conisderados públicos (serão criados em “www/data/files”). Caso a configuração seja desfeita, os arquivos públicos serão removidos de “www/data/files”.
 

Observação

 

O Lumis Portal oferece como sugestão de montagem deste serviço um prebuilt channel com uma estrutura de canal, páginas, e disposição das interfaces nas páginas.

Para informações sobre como instanciar canais, páginas, e atribuição de permissões no controle de acesso, consultar a seção sobre Portal Studio .

Através do right-click da interface, encontram-se disponíveis parametrizações para a instância de serviço e customizações para a interface. Consultar a seção Customização de instâncias de Interfaces.

Para o serviço de Imagens, o XSL é customizado e exibe campos fixos. Portanto, a habilitação e desabilitação de campos não refletem na interface administrativa.

O Repositório de Mídias depende do mecanismo de transformação de arquivos para possibilitar a reprodução de videos/audio cujo o formato não seja compatível com todos os browsers. Entretanto é importante que o usuário valide se o mecanismo de transformação gera transformações que sejam reproduzidas nos browsers desejados, isto pois, o mecanismo de transformação de arquivos utiliza a biblioteca Jave que ao converter video/audio de um formato para outro pode não conseguir fazer uma conversão adequada. Neste caso, o usuário poderá utilizar um transformador customizado que faça uso de uma outra biblioteca que o atenda.
