Administração do Serviço

 Top  Previous  Next

Como o serviço Envie para um Amigo é tipicamente de colaboração, não possui o objetivo de publicação de conteúdo, não havendo portanto o personagem “editor de conteúdo”. No entanto, há o personagem “administrador do serviço”, que fica responsável pelo conteúdo e formato da mensagem a ser recebida pelo destinatário.

O administrador do serviço deve ter a permissão “Administrar” para ter acesso ao ambiente administrativo do serviço. Por definição, o administrador de serviço também possui a permissão de “Postar Mensagem”.

As mensagens enviadas pelos usuários aos seus “amigos” são listadas na área administrativa, podendo também ser excluídas.

Cada registro da mensagem enviada para um amigo é linkável para os detalhes da mesma. Os detalhes incluem: Nome do amigo, e-mail do mesmo, o remetente, e-mail do remetente, o link da página e o título.

Como exemplo, ao clicar no segundo registro da mensagem na área administrativa, a página de detalhes da mensagem é exibida (como pode ser visto na figura abaixo).

O formato e disposição do conteúdo da mensagem do e-mail destinado ao “amigo” podem ser configurados a partir da área administrativa, acionando “Configurar”.

No formulário “Configuração” exibido, é apresentado um conteúdo default embaixo de cada campo. Este conteúdo mistura texto com formatações HTML e tags especiais que recuperam informações disponibilizadas pelo usuário no formulário "Enviar Mensagem".

Para compor o formato e disposição do conteúdo do e-mail, o administrador do serviço pode arrastar as tags para os campos e mesclar com formatações HTML. Caso contrário, se o campo não for preenchido, será adotada a configuração default.

Os campos que são configurados e que fazem parte do conteúdo são:

•E-mail do remetente : E-mail localizado no campo “De:” de uma caixa-postal do destinatário. O administrador pode configurar com um e-mail fixo ou adotar o e-mail que é preenchido pelo usuário no formulário “Enviar Mensagem”. Para recuperar o e-mail do remetente, deve-se utilizar a tag $lum_fromEmail$. A configuração default adotada para este campo é o e-mail do framework do portal configurado no Gerenciador de Mensagens;
•Assunto: Tópico da mensagem localizado no campo assunto da caixa-postal do destinatário. Pode-se preencher com uma expressão ou fazer com que o título da página localizado na barra de título do browser seja exibido como assunto da mensagem. Para isso, deve-se utilizar a tag $lum_linkTitle$;
•Corpo: Refere-se à mensagem propriamente dita, recebida pelo destinatário. O administrador pode utilizar tags que recuperam o nome do remetente $lum_fromName$, bem como seu e-mail $lum_fromEmail$; e o nome do destinatário $lum_toName$; que são informados pelo usuário no formulário “Enviar Mensagem”. Pode-de também utilizar a tag $lum_link$ que recupera o endereço da página. Esta tag pré-definida do Lumis deve ser utilizada entre tags HTML que fazem referência. Por fim, pode-se utilizar a tag $lum_message$ que recupera o comentário informado pelo usuário no formulário “Enviar Mensagem”. Esta última tag pode ser utilizada juntamente com a tag HTML <pre> para que sejam preservados os espaços utilizados no comentário. Textos podem ser utilizados juntamente com as tags especiais para compor o conteúdo da mensagem, como por exemplo: “Comentário do Remetente”; “O seguinte link foi enviado por”.
