Cache de XSL

 Top  Previous  Next

O cache  de XSL é automático. Todos XSL registrados no portal são cacheados na memória, a partir do primeiro acesso a objetos que os utilizam. Assim, ocorrerá um acesso mais rápido a esse XSL no momento da renderização da interface que o utiliza.

 

Ações disponíveis no gerenciador de cache de XSL:
 

•Habilitar Cache: Habilita o cache de XSL. O XSL será cacheado a partir do primeiro acesso a um objeto que utiliza tal estilo;
•Limpar Cache: Permite retirar o conteúdo XSL do cache de memória. Uma vez que tal conteúdo XSL é acessado, é automaticamente cacheado;
•Desabilitar Cache: Os arquivos XSL deixam de ser cacheados a não ser que o administrador os habilite novamente;
•Excluir: Remove o XSL da lista do gerenciador de cache e conseqüentemente do cache de memória. Porém, a partir do próximo acesso a uma interface, ele será novamente cacheado.
