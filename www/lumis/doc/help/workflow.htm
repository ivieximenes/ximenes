Customizações de workflow são implementadas em arquivos de definição XML. Estes arquivos, obrigatoriamente, devem ser nomeados como workflowdefinition.xml e devem estar localizados em diretórios criados especificamente para customizações, a partir de “lumisdata/def”.

Observação

 

Maiores informações, consultar a seção Customização do Workflow Padrão .

Para registrar um novo workflow, basta acionar Adicionar na área administrativa de Workflows do Gestor de Conteúdos.

Logo em seguida, deve-se especificar o caminho do arquivo de definição, sem o workflowdefinition.xml, já que todos possuem o mesmo nome.  A figura abaixo exemplifica um registro de um workflow para FAQ. O arquivo de definição foi criado anteriormente em "lumisdata/def/Corporate/Workflow". Para registrá-lo, basta informar o caminho pertencente a "lumisdata/def".

Caso haja alterações nas definições do workflow customizado, é necessário atualizar o registro do arquivo, neste mesmo ambiente. Para isso, seleciona-se tal arquivo e logo em seguida, aciona-se Atualizar.

Caso determinado workflow não deva fazer parte do portal e não mais utilizado pelos serviços, pode-se excluí-lo da área administrativa de workflow. No entanto, o arquivo continua existindo fisicamente no projeto.

Se o ID do workflow for alterado, um novo workflow será registrado se o atual for removido.

Arquivo de Definição do Workflow Padrão

O workflow padrão, que é utilizado pelos serviços do Lumis, está localizado em “/lumis/workflow/standard” e não deve ser alterado.
