O serviço pode ser montado de diversas maneiras, para o administrador do serviço, dependendo de como foi arquitetado pelo administrador do portal.

Para utilizar o serviço Robots.txt, basta instanciá-lo em um canal e arrastar suas interfaces para as áreas das páginas em modo de edição. Suas interfaces correspondem a um ambiente administrativo de cadastro de informações de user-Agents para o robots.txt e pré-visualização do mesmo.

 

•Administração de Robots.txt: Esta interface exibe a lista de configurações de user-Agents que compõem o Robots.txt;
 
•Adicionar: Essa interface é acessada a partir da interface Administração de Robots.txt. Nela, os parâmetros e critérios para cada user-Agent são cadastrados para o Robots.txt;
 
•Editar: Essa interface é acessada a partir da interface Administração de Robots.txt. Nela, os parâmetros e critérios para cada user-Agent, podem ser editados, conforme necessário;
 
•Administração de Sitemap: Essa interface permite informar qual(is) sitemaps irão compor o robots.txt. Os arquivos acessados através da URL informada nessa administração, deve ter sido cadastrado no serviço Sitemap;
 
•Visualizar Robots.txt: Esta interface permite uma pré-visualização na página das informações de configurações de parâmetros e critérios cadastradas para o robots.txt.

 

Para disponibilizar o serviço de Robots.txt basta instanciar as interfaces em páginas de um canal restrito do portal, onde apenas o administrador do serviço terá acesso, após o login. A fim de proporcionar uma maior organização no portal, é aconselhável que tais páginas pertençam a um canal específico, como por exemplo “Robots.txt”.

 

Atribuição de Permissões

Para que o editor de conteúdo possa acessar o ambiente administrativo de robots.txt, o administrador do portal deve atribuir a ele a permissão "Administrar". Esta permissão é concedida no controle de acesso da instância do serviço.
