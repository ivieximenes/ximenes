Nesta seção estão os passos necessários para a instalação do banco de dados do Lumis Portal nos servidores Microsoft SQL Server 2012 e Microsoft SQL Server 2014:

 

1) Execute o arquivo /setup/generatesql.cmd, para o Windows®, ou generatesql.sh, para o Linux, para criar os arquivos de script do banco de dados.

 

2) Abra o SQL Server Management Studio, crie um novo database e execute o script db_sqlserver.sql.

 

3) Habilite o nível de isolamento SNAPSHOT no database criado executando o seguinte SQL:

 

 ALTER DATABASE <nome_database> SET READ_COMMITTED_SNAPSHOT ON;

 

4) No arquivo lumisdata/config/lumishibernate.cfg.xml altere as informações de conexão com o banco de dados.

 

<!-- MS SQL Server 2012 e 2014 -->

<property name="dialect">org.hibernate.dialect.SQLServerDialect</property>

<property name="connection.driver_class">net.sourceforge.jtds.jdbc.Driver</property>

<property name="connection.username">DB_USERNAME</property>

<property name="connection.password">DB_PASSWORD</property>

<property name="connection.url">jdbc:jtds:sqlserver://HOST_NAME:1433;databaseName=DATABASE_NAME;instanceName=INSTANCE_NAME;useLOBs=false</property>

 

Uma outra alternativa de conexão com o banco de dados, é a configuração do datasource no servidor de aplicação.

 

5) Execute o arquivo /setup/initializeportal.cmd, para o Windows®, ou initializeportal.sh, para o Linux. Este script executa uma aplicação Java que abre uma conexão com o banco de dados, utilizando as informações do arquivo lumisdata/config/lumishibernate.cfg.xml. Este script também insere os dados iniciais necessários para o funcionamento do Lumis Portal.

 

Observação

É importante ressaltar que o protocolo TCP/IP não é habilitado automaticamente pelo Microsoft SQL Server. Portanto, habilitar tal protocolo caso esteja ocorrendo erro de conexão do Lumis Portal com o banco.
