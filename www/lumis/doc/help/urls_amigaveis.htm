Urls Amigáveis

 Top  Previous  Next

O cache possui o mesmo comportamento independente se a estrutura está configurada para utilizar urls amigáveis ou não. A diferença está no nome do arquivo e aonde ele é salvo.

 

Como foi visto na seção anterior, o cache de página (página estática) é exibida com extensão ".HTM". Além disso, o nome do arquivo é um Guid. Como exemplo: http://dominio:porta/contexto/data/pages/8A488A1D22BD37260122BD51809F0652.htm. A página htm é gerada emwww/data/page.

 

Porém, quando para a estrutura cacheada também está configurada para ser exibida com urls amigáveis, o nome do arquivo será aquele definido no formato do canal ou página, e o arquivo htm será gravado no path determinado pela configuração do formato de geração de urls amigáveis.

 

Portanto, torna-se transparente para o usuário se a página estática ou dinâmica já que uma estrutura configurada para utilizar urls amigáveis exibirá páginas com extensão HTM independente de estar cacheada ou não.

 

O caminho do arquivo de cache de página HTML, relativo à raiz da aplicação web do portal, é exibido no Gerenciador de Páginas em Cache

 

 

Observação

 

Se cache HTML for habilitado para determinada estrutura configurada para exibir com url amigável, os diretórios/arquivos correspondente aos caminhos das urls amigáveis serão gerados. Pode ser desejado não misturar estes arquivos de cache com outros arquivos.
