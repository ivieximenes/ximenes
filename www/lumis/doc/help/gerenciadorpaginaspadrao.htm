Páginas Padrão

 Top  Previous  Next

Como é sabido, em um portal é possível construir várias soluções. Imagine um portal que contém várias soluções correspondentes a websites de uma empresa.

 

Cada uma delas deve conter uma página padrão principal e uma página padrão de login. Quando a URL da solução for acessada, a solução será redirecionada para a página padrão principal. Por sua vez, quando o usuário acessar uma página ao qual ele não tem acesso, será exibida a página padrão de login.

 

Nada impede também que todas as soluções apontem para a mesma página padrão principal e mesma página padrão de login. Ou também que todas as soluções apontem para a mesma página padrão principal e página padrão de login diferente, ou vice-versa. Em uma mesma solução ainda, cada área pode ter a sua própria página de login.

 

Enfim, a forma como tais páginas serão cadastradas depende a arquitetura das soluções.

Para cadastrar uma página padrão principal ou de login, basta clicar em Adicionar e preencher as seguintes informações:


 

•Adicionar: Permite incluir uma nova página padrão;

 

Aba geral

oNome: Nome da página padrão. Este nome será exibido na lista de administração de páginas padrão e poderá ser realizadas buscas pelo mesmo;
oWebsite: Permite especificar qual website a pagina padrão se refere. Quando o website for acessado por uma determinada URL, serão utilizadas as páginas de Home Page e/ou Login que forem configuradas;

o Página: Para especificar qual página do portal será exibida como padrão (uma para home page, outra página de login) selecioná-la através do botão ;

o Tipo: Permite definir se a página selecionada no campo acima, será padrão para página de login ou para página principal da solução;

o Página de Login: Define a página como tipo de login, significa quando um usuário acessar uma área que não possui acesso, a página configurada como padrão de login será exibida;

o Home page: Define a página como tipo home page (página principal) quando a URL da solução for acessada; e após expiração da sessão e login;

o Canal: Pode-se especificar a abrangência da exibição da página padrão de login. A abrangência corresponde ao canal especificado ao seu último canal filho. Caso este canal seja definido, esta página padrão não será utilizada em contextos que não possuam canal. (exemplo: login.jsp)

oPrioridade: Ordem em que as páginas padrão (login ou Home Page) são verificadas para atender a uma requisição ao portal. Páginas padrão com valor menor de prioridade são utilizadas primeiras. Ou seja, se houver mais de uma página padrão que possa ser utilizada para o redirecionamento, a página padrão com menor valor de prioridade será utilizada.

 

Observação: Todos os campos desta aba, exceto o canal, são obrigatórios.

 

Aba Regras de Contexto

oRegras de Contexto: Permite especificar quais capacidades um determinado dispositivo deve possuir para ser redirecionado para a URL especificada. Neste campo podem ser introduzidas expressões lógicas como AND, OR, ==, != , >, <, >=, <= e parênteses. Para valores do tipo string utilizar aspas simples conforme apresentado nas figuras anteriores.

 

Aba Regras de URL

oRegras de URL: Permite especificar para quais URLs do website será aplicado o redirecionamento da página padrão.

 

 

•Editar: Permite alterar as informações da página padrão, como a especificação da URL, por exemplo, ou informar outra página padrão, mantendo-se ou não a URL;
•Excluir: Permite excluir a página padrão do gerenciador. No entanto a página não será excluída do portal. A partir daí será exibida a página padrão do portal (principal ou de login).

Observações

 

1. Lembrar de dar acesso ao usuário guest a nova página de login e negar as permissões de acesso a este mesmo tipo de usuário aos canais subseqüentes, caso a herança entre os canais esteja mantida;

2. Após login na página cadastrada como padrão, o portal redirecionará para a página principal também padrão, ou aquela que já vem com o produto (esta é considerada como principal quando não há outra página cadastrada como principal padrão).

  No caso de login realizado em página não definida como padrão, será realizado apenas um refresh na página.
