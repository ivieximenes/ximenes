O serviço pode ser montado de diversas maneiras, dependendo de como foi arquitetado pelo administrador do portal.

Para utilizar o serviço Agenda de Sala de Reunião, basta instanciá-lo em um canal e arrastar suas interfaces para as áreas das páginas em modo de edição. Suas interfaces correspondem a um ambiente administrativo de cadastro e agendamentos de recursos, e áreas de visualização do serviço pelo usuário final.

 

•Administração de Recursos: Esta interface permite o cadastro e manutenção dos recursos que podem ser reservados;
•Administração de Agendamentos: Esta interface permite cadastrar os agendamentos, informando a sala e, facultativamente, outros recursos, estabelecendo o período que estarão reservados. Os agendamentos cadastrados formam uma lista que é exibida nesta mesma interface, para possíveis manutenções;
•Agendamentos: Esta interface somente exibe os agendamentos realizados e mantidos no cadastro;
•Agenda diária: Esta interface exibe um calendário com todos os agendamentos do dia selecionado;
•Detalhes: Esta interface exibe todas as informações de cada agendamento selecionado a partir das interfaces "Administração de Agendamentos" e "Agendamentos".

Para disponibilizar o serviço para o usuário final, basta instanciar a interface Agendamentos em uma página.O ambiente administrativo requer uma página para a Administração de Recursos e outra para Administração de Agendamentos, onde, respectivamente, apenas os administradores do serviço e os usuários com permissão de realizar agendamentos, terão acesso. A fim de proporcionar uma maior organização no portal, é aconselhável que esta página pertença a um canal específico, o qual está sendo chamado aqui de “Agenda de Sala de Reunião”.

Atribuição de Permissões

O serviço Agenda de Sala de Reunião trabalha com diferentes perfis de utilização do serviço que podem ser atribuídos aos usuários no controle de acesso da instância de serviço:

•Administrar: Usuários com esta permissão podem cadastrar recursos e realizar agendamentos, bem como excluir os agendamentos realizados por quaisquer usuários. Esta permissão inclui também a permissão “Agendar”;
•Agendar: Usuários com esta permissão podem realizar agendamentos de salas e outros recursos. É concedido a eles o direito de excluir somente os seus agendamentos;

Observação

 

O Lumis Portal oferece como sugestão de montagem deste serviço um prebuilt channel com uma estrutura de canal, páginas, e disposição das interfaces nas páginas.

Para informações sobre como instanciar canais, páginas, e atribuição de permissões no controle de acesso, consultar a seção sobre Portal Studio.

Através do right-click da interface, encontram-se disponíveis parametrizações para a instância de serviço e customizações para a interface. Consultar a seção Customização de instâncias de Interfaces. Entre elas o repositório de imagens a ser utilizado no editor HTML.
