Nesta seção estão os passos necessários para a instalação do banco de dados do Lumis Portal no Servidor MySQL:

 

1) Execute o arquivo /setup/generatesql.cmd, para o Windows®, ou generatesql.sh, para o Linux, para criar os arquivos de script do banco de dados.

 

2) Abra o MySQL Query Browser, crie um novo database e execute o script db_mysql.sql.

 

3) No arquivo lumisdata/config/lumishibernate.cfg.xml altere as informações de conexão com o banco de dados.

 

<!-- MySQL 5.5 e 5.6 -->

<property name="dialect">org.hibernate.dialect.MySQL5InnoDBDialect</property>

<property name="connection.driver_class">com.mysql.jdbc.Driver</property>

<property name="connection.username">DB_USERNAME</property>

<property name="connection.password">DB_PASSWORD</property>

<property name="connection.url">jdbc:mysql://HOST_NAME/DATABASENAME?characterEncoding=UTF-8&amp;useConfigs=maxPerformance</property>

<property name="connection.isolation">2</property>

 

Uma outra alternativa de conexão com o banco de dados, é a configuração do datasource no servidor de aplicação.

 

4) Execute o arquivo /setup/initializeportal.cmd, para o Windows®, ou initializeportal.sh, para o Linux. Este script executa uma aplicação Java que abre uma conexão com o banco de dados, utilizando as informações do arquivo lumisdata/config/lumishibernate.cfg.xml. Este script também insere os dados iniciais necessários para o funcionamento do Lumis Portal.

 

 

Observações

 

O driver MySQL Connector Java não está incluído na distribuição do Lumis Portal, ele pode ser encontrado no link abaixo:

http://dev.mysql.com/downloads/connector/j/

 

Verificar se a versão do conector do MySQL utilizado, corresponde ao especificado no arquivo setup/setclasspath.cmd ou setup/setclasspath.sh . Caso contrário, alterar a versão do conector no arquivo setclasspath .

Deve-se ainda configurar o default charset para UTF-8 para que funcione com STRICT MODE.

As configurações do MySQL exigidas pelo Lumis Portal podem ser passadas na inicialização ou configuradas no arquivo my.ini:

•default-character-set=utf8;
•sql_mode=STRICT_TRANS_TABLES

 

1.Existe uma limitação quando utilizado o MySQL que a data não pode ser inferior a 01/01/1753. Quando isso ocorre, é retornado uma mensagem de erro informando que a data é inválida.
2.Existe uma configuração no MySQL que limita o tamanho máximo do pacote recebido pelo servidor que pode causar problema em cenários de cluster. Para contornar tal limitação, recomenta-se configurar a variável do servidor max_allowed_packet, conforme no exemplo abaixo:

[mysqld]

max_allowed_packet=16M

 

Onde 16M é o tamanho máximo do pacote. O padrão do servidor é 1M. O valor deve ser ajustado de acordo com a necessidade da aplicação.
