Arquivos com cópia única

 Top  Previous  Next

Essa implementação de sistema de arquivos mantém uma cópia única dos arquivos no sistema de arquivos do portal. Uma alteração no sistema de arquivos (adição, edição e remoção) gerada em um nó do cluster não é replicada nos outros nós, uma vez que o sistema de arquivos é compartilhado entre os nós do cluster.

 

Exemplo:

Imagine que um arquivo do sistema de arquivos do portal é escrito no nó Server1, e que posteriormente esse mesmo arquivo seja lido nos outros nós do cluster. Nesse caso, o arquivo seria escrito somente pelo nó Server1 e os acessos a esse arquivo são feitas em pastas locais, como mostrado na figura abaixo:

 

 

Preparação do sistema de arquivos

Essa implementação de sistema de arquivos requer que os arquivos estejam localizados em pastas compartilhadas entre os nós do cluster, como na lista abaixo:

•def: a sua raíz deve ser a pasta <lumisdata>/shared/def
•data: a sua raíz deve ser a pasta <lumisdata>/shared/data
•source: a sua raíz deve ser a pasta <lumisdata>/shared/source
•diretórios estáticos de websites: devem estar localizados no mesmo local configurado no Gerenciador de Websites.
