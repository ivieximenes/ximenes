Ao criar página:

 

•Nome: Nome da página a ser exibida na estrutura hierárquica de canais e página e no menu do portal;
•Descrição: Descrição da página para facilitar sua administração. O montador utiliza esse valor para montar a meta-tag description a ser exibida no header da página html e utilizado pelos mecanismos de busca externos;
•Título: Título da página que será exibido na barra de título do browser. Esse valor corresponde à tag <title> do header do html a ser utilizado pelos mecanismos de busca externos. Quando não preenchido, o montador utilizará o valor informado no campo Nome;
•Palavras chave: Palavras que definam ou façam parte do conteúdo da página. Estas palavras serão usadas por mecanismos de busca da Internet para que os mesmos possam encontrar e listar o site em seus resultados. Devem ser separadas por vírgulas;
•Gerar Arquivo HTML: Permite determinar se a página será cacheada apenas para usuário guest, usuários cadastrados, ou se irá herdar o que está definido para o canal da página.

 

Nos casos de personalizações e entrada de dados, pode-se cachear a página apenas para usuário Guest . Dessa forma, para os usuários cadastrados, será exibida sempre a página dinâmica.

Observações

 
•A página a ser cacheada não deve conter interfaces com ordenação e filtros, por exemplo, e nem interfaces que exigem interação com o usuário;
•O cache da página ficará desabilitado, não explicitamente, caso seja configurado para o página, ou por herança, utilizar o protocolo HTTPS.

 

•Centralizar Páginas: Permite definir se todo o layout  da página será centralizado ou não, ou se essa propriedade será herdada do canal “pai”. O default do portal é não centralizar as páginas;

 

•Tipo de Layout: Permite definir como o código HTML da página será gerado. As opções são:

 

1.Herdar Configuração: utiliza o valor definido para o canal pai.
2.Com tabelas: o código será gerado utilizando a tag html <table>.
3.Sem tabelas: o código será gerado utilizando a tag html <div> .
4.Arquivo de layout: o código será gerado a partir do arquivo de layout informado. O caminho do arquivo utilizado é relativo à raiz do classpath ou à pasta "lumisdata/shared/def". O arquivo de layout também pode estar em um módulo ou um tema. Para mais detalhes consulte o tópico Layout File. É sugerida uma lista de arquivos de layout que estejam na pasta www dos temas aplicados.

 

O valor default do framework  do portal é gerar código html com tabela.

 

É importante salientar que em páginas tableless o que é visualizado fora do modo administrativo do portal (fora do F2) pode ficar diferente do que é visualizado dentro do F2.
 

•Posição: O local absoluto da página no navegador e na  Barra de Navegação. A primeira posição é representada por 0 (zero). Caso haja mais de uma página com a posição zerada, são ordenadas alfabeticamente no navegador e na Barra de Navegação exibida para o usuário final da solução;

 

•Oculto: Determina se a página estará oculta ou não na  Barra de Navegação. Essa opção deve ser usada para páginas que só devem ser acessadas por links a partir de outras páginas;

 

•Copiar Página (Propriedade visível na criação da página): Permite que durante a criação da página ela seja especificada a partir de uma outra página existente. Alterações posteriores na página copiada não serão repassadas;

 

•Mover para Canal: Permite especificar o canal destino da página, não deixando réplicas;

 

•Aplicar Template: Permite que a página seja criada a partir de um template de página. Desta forma, todas as propriedades utilizadas pelo template e as interfaces nele instanciadas serão repassadas para a página que possui tal template aplicado. Para desassociar o template da página, basta clicar no "x" de excluir, e a página não receberá as atualizações do template, mantendo-se as configurações que foram atribuídas ao canal com a aplicação do template.

 

Observação

 

Caso o canal da página já tenha um template padrão atribuído, este portanto virá automaticamente preenchido no campo “Aplicar Template”.
