O serviço pode ser montado de diversas maneiras para os editores de conteúdo e para o usuário final da solução, dependendo de como foi arquitetado pelo administrador do portal.

Para utilizar o serviço, basta instanciá-lo em um canal e arrastar suas interfaces para as áreas das páginas em modo de edição. Suas interfaces correspondem a um ambiente administrativo de cadastro de álbuns e fotos e áreas de visualização do serviço pelo usuário final.

 

•Administração de Álbuns: Esta interface permite o cadastro e manutenção de álbuns. Permite também que cada álbum cadastrado seja linkável para o ambiente administrativo de suas fotos;
•Administração de Fotos: Esta interface permite o cadastro e manutenção de fotos pertencentes a um álbum selecionado na interface administrativa de álbuns;
•Lista de Álbuns: Lista os álbuns cadastrados, aprovados pelo processo de workflow e de acordo com o período de publicação;
•Lista de Fotos: Lista as fotos cadastradas para cada álbum, correspondentes ao campo “Miniatura” na administração de fotos. Caso não haja uma miniatura cadastrada, ele utilizará a imagem original, dando preferência para uma transformação com chave "scaling_image_sm" caso exista;
•Detalhes de Álbum: Esta interface exibe informações sobre álbum quando este é selecionado pelo usuário para visualização;
•Foto: Esta interface exibe a foto que foi cadastrada no campo “Foto” da administração de fotos. Cada foto exibida é selecionada a partir da lista de fotos.

Para disponibilizar o serviço de Álbum de Fotos para o usuário final, basta instanciar as interfaces Detalhes do Álbum, Foto, Lista de Álbuns e Lista de Fotos em uma mesma página do portal. Além disso, instancia-se outra interface Lista de Álbuns em uma outra página para que se tenha acesso à página anterior.

O ambiente administrativo de álbuns e fotos requer duas páginas em um canal restrito do portal, onde apenas os editores de conteúdo terão acesso, após o login. A fim de proporcionar uma maior organização no portal, é aconselhável que estas páginas pertençam a um canal específico, o qual está sendo chamado aqui de “Álbum de Fotos”.

 

 

Atribuição de Permissões

Para que o editor de conteúdo possa cadastrar/editar e publicar álbuns e fotos, o administrador do portal deve atribuir a ele a permissão “Editar conteúdo de instância de serviço”. Esta permissão é concedida no controle de acesso da instância do serviço.

Na figura a seguir, há um exemplo do controle de acesso de uma instância de serviço, cujo usuário “Carla” está com permissão “Editar Conteúdo”, atribuída pelo administrador do portal. Com a permissão “Editar Conteúdo” o usuário tem acesso a interface administrativa porém não conseguirá cadastrar conteúdo. Para isso deve ser concedido pelo menos um dos papéis do workflow padrão. Portanto foi concedido ao usuário “Carla”, o papel “Editor”.

Para que o editor de conteúdo realize upload de arquivos, é necessário que ele tenha a permissão de Efetuar Upload de arquivo no Gerenciador de acesso do portal.
 

Observação

 

O Lumis Portal oferece como sugestão de montagem deste serviço um prebuilt channel com uma estrutura de canal, páginas, e disposição das interfaces nas páginas.

Para informações sobre como instanciar canais, páginas, e atribuição de permissões no controle de acesso, consultar a seção sobre Portal Studio .

Parametrizações da Instância de Serviço

A propriedade “Propriedades de Gerenciamento de Arquivos” acessada através do right-click das interfaces administrativas, permite determinar se as imagens dos álbuns serão públicas ou privadas.

Caso os arquivos sejam públicos, as interfaces de outros serviços que realizam download de arquivos do repositório de imagens, terão acesso direto, sem autenticação do usuário (os arquivos estarão localizados em “www/data/files”). Caso contrário, se os arquivos forem privados, estarão em uma área restrita (acessados via jsp), havendo necessidade de autenticação pelo portal.

Uma vez que o repositório seja configurado como público, os aquivos já existentes bem como os que forem adicionados após tal configuração, será conisderados públicos (serão criados em “www/data/files”). Caso a configuração seja desfeita, os arquivos públicos serão removidos de “www/data/files”.

 

Customização da Instância de Interface

Através da opção Estilo no right-click sobre a interface Lista de Álbum é possível configurar as formas de exibição dos menus, além do estilo Default:

•Combo seleção: Listas as opções de álbuns visualizados pelo usuário final em uma combo.

•Combo seleção com pop-up: Possui a mesma finalidade do Estilo anterior. Porém, quando o usuário selecionar um álbum, o mesmo será aberto em um pop-up.
 

Observação

 

Através do right-click da interface, encontram-se disponíveis parametrizações para a instância de serviço e customizações para a interface. Consultar a seção Customização de instâncias de Interfaces.
