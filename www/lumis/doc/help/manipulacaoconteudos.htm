Manipulação de Conteúdos

 Top  Previous  Next

Um conteúdo de uma instância é composto por dados e metadados. Os dados mais comuns são Título, Introdução e Conteúdos. Outros campos podem compor o conteúdo, dependendo do serviço. Já os metadados são informações complementares ao conteúdo tais como data/hora de publicação, workflow, comentários, entre outros.

 

•Barra de comandos: A interface administrativa contém as seguintes funcionalidades: Adicionar, Editar, Excluir e Visualizar. Porém, dependendo do serviço, esta barra pode conter outros comandos, tal como Versões que permite determinar que versão do conteúdo será publicado;

 

•Filtros: Campos cujos valores  aplicados à lista de conteúdos cadastrados, determinam os registros a serem retornados. Um filtro é composto por um campo ou por uma combinação deles;

 

Por default, o operador do filtro está definido como “igual a”. Isto significa que os resultados a serem obtidos devem ser exatamente igual ao valor digitado. Pode-se filtrar por partes iniciais do conteúdo, digitando "string*" ou "string%"; por finais, digitando "*string" ou "%string";e por partes centrais, digitando "*string*" ou "%string%";

 

•Paginação: Quando uma lista ultrapassa determinada quandidade de registros estipulados para ser exibidos na interface administrativa, é gerada uma paginação dos mesmos. O limite default de registro de conteúdos por página é 50. A figura abaixo exemplica essa funcionalidade. A interface foi configurada para exibir dois conteúdos, se o serviço apresenta mais, é gerada uma paginação, com cada página apresentando dois conteúdos. A última página, porém, pode apresentar até dois conteúdos;
