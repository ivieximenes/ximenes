O serviço de Links possui um ambiente administrativo para que o editor de conteúdo cadastre links e categorias. As categorias cadastradas possuem o papel de agrupamento para o links cadastrados e exibidos no portal. Uma vez cadastrados links ou categorias, pode-se re-editá-los ou excluí-los.

Administração de Categorias

Para o cadastro de uma categoria basta acionar o comando Adicionar e informar o nome da categoria a ser criada, a qual pode ser alterada posteriormente através do comando Editar, caso seja necessário.

Administração de Links

Para o cadastro de links, devem ser preenchidas as seguintes alterações.

Informações básicas:

•Nome: Título assumido pelo link (campo obrigatório);
•Introdução: Pequena descrição do conteúdo do link;
•Imagem: Imagem que pode estar associada ao conteúdo do link carregada a partir do repositório de imagens do portal;
•Categoria: Associar o link a uma das categorias já cadastradas. Pode-se também não associar a nenhuma delas.
•Prioridade: Valor atribuído ao link para efeito de ordenação, filtro, regras de publicação.

Propriedades do Link:

•Tipo do Link: Conteúdo Interno, URL, Página Interna e Documento

Caso o tipo de link seja cadastrado como “conteúdo interno”, é necessário editar este conteúdo para o qual o usuário será remetido. Para isso é utilizado o editor de texto/HTML com as formatações devidas.

Caso o tipo de link seja cadastrado como “página interna”, é necessário escolher a página da estrutura de canais para qual o usuário será remetido.

A vantagem de usar essa opção é que não é necessário saber a URL para uma página interna do portal.

Uma vez que tal página não pertença mais ao portal, ela será automaticamente desconectada do campo Página.

Caso o tipo de link seja cadastrado como “URL”, é necessário inserir a URL da página que redirecionará o usuário para páginas que não fazem parte da solução. O campo URL, de preenchimento obrigatório, já vem com a informação do protocolo (http://). Mas outros podem ser utilizados (https://, mms://, rtsp://, ftp://). A URL também pode ser relativa, e, neste caso, ela será relativa ao endereço base do portal sendo acessado e não deve ter o protocolo informado. Além disso, o editor de conteúdo pode determinar se a página será exibida em pop-up ou não. Caso seja exibida em um pop-up, pode-se ainda configurar por exemplo, largura e altura do pop-up.

Caso o tipo de link seja cadastrado como “Documento”, é necessário atribuir o nome do arquivo escolhido a partir do repositório de documentos do portal. Desta maneira o usuário da solução, ao clicar neste tipo de link, fará o download do arquivo.

O ícone ao lado do campo abre um pop que permite carregar o documento que se deseja associar ao link no portal.

O serviço de Links permite que o editor de conteúdo estabeleça regras de publicação como período que determinado link estará disponível para visualização do usuário final. Permite também publicar o conteúdo para determinados grupos e ou usuários, para outras instâncias do serviço de Links e publicar o conteúdo em outros idiomas. Permite também que conteúdos sejam destaques dentro de um período pré-estabelecido.

Por ser um serviço de publicação de conteúdo, apresenta funcionalidade de workflow para aprovação de cada link cadastrado.

Os links cadastrados formam um conjunto de registros para posteriores manutenções em seus conteúdos. A disponibilização dos campos que aparece na figura abaixo depende de como o administrador do portal configurou o ambiente administrativo.

Observação

 

Para informações sobre período de publicação, workflow, publicação para outros idiomas e visualização do conteúdo, consultar a seção “Funcionalidades Gerais dos Serviços de Gestão de Conteúdo”.
