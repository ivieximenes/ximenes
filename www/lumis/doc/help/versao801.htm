Versão 8.0.1

 Top  Previous  Next

A versão 8.0.1 possui todas as modificações incluídas até a versão 8.0.0. Abaixo seguem os destaques das demais funcionalidades não existentes na versão 8.0.0.

Melhorias

•O processo de deploy de Módulos editáveis agora abre a interface de execução de processos interativos para acompanhamento do processo de deploy.

Correções

•Correção no editor de estilo ao adicionar um estilo em um módulo que contenha entidade Hibernate;
•Correção no editor de estilo para a operação de Salvar também limpar o cache do estilo;
•Correção no editor de estilo para a operação de Salvar também registrar o tema ou fazer deploy do módulo do estilo adequadamente;
•Correção no editor de estilo para a operação de Salvar também procurar o estilo a ser copiado nos temas;
•Correção no Portal Studio nas propriedades de canal na exibição do campo Template de página padrão;
•Correção no Portal Studio ao tentar editar tema cujo identificador inicia com br.com.lumis ou com.lumis;
•Correção no Portal Studio na configuração de controle de acesso de instância de serviço, quando nome de permissão customizada do serviço conflita com nome de papel de workflow aplicado na instância de serviço;
•Correção no Portal Studio que exibia tela de erro quando sua URL raiz era acessada diretamente por usuário não autenticado;
•Correção no Portal Studio na tela de edição de filtros de uma interface, onde a opção de editar um filtro não estava disponível;
•Correção no Portal Studio na exibição dos overlays de interfaces que não apareciam corretos imediatamente;
•Correção na atualização de componente que executava passos do script de atualização já executados;
•Correção no processo de deploy de componente que não copiava corretamente os seus arquivos em www;
•Correção na atualização de módulo que contenha classes que implementam IService ou IServiceInterface, onde as antigas continuavam sendo utilizadas;
•Correção no processo de deploy de módulo para não permitir instalação de um mesmo componente em mais de um módulo;
•Correção no processo de deploy de módulo no tratamento de alguns cenários onde um passo ao ser executado resultava em erro e não disponibilizava opção de ignorar o passo;
•Correção no processo de deploy de módulo para expirar estilos contidos nele;
•Correção no processo de deploy de módulo em ambiente com cluster habilitado, que não levava em conta corretamente classes REST ou entidades Hibernate presentes;
•Correção no processo de deploy de um módulo editável que sobrescrevia um módulo não editável;
•Correção no processo de deploy em cenários onde o módulo continha arquivos com espaço no nome;
•Correção no registro de serviço que não expirava cache de XSL de interfaces que não estão montadas em página;
•Correção na instalação de componente, ao gerar commando de criação de tabela a partir de databasedefinition.xml para banco de dados MySQL, para explicitar o uso de InnoDB;
•Correção no editor de temas e módulos quando nome de arquivo possui caractere não alfanumérico;
•Correção no editor de temas e módulos, onde o menu de contexto na árvore de pastas e arquivos não era sempre exibido no Internet Explorer;
•Correção no gerenciador de temas que exibia a opção Editar tema para temas não editáveis;
•Correção no editor de temas ao efetuar upload de zip onde os arquivos adicionados durante o upload nem sempre refletiam na solução até que o módulo fosse atualizado;
•Correção no Gerenciador de módulos onde após desinstalar um módulo algumas operações não funcionavam sem acessar novamente o gerenciador;
•Correção no Gerenciador de módulos na interface de detalhes de módulo para não exibir opção de reprocessar quando o botão de atualizar não está presente;
•Correção no Assistente de Criação de Serviços de Conteúdo que incluía o elemento desnecessário <serviceIsolation/> em sources sem opção de publicar para instâncias habilitada;
•Correção no serviço de Conteúdo Hierárquico que permitia cadastro de conteúdos que provocasse ciclo na hierarquia;
•Correção no serviço de Álbum de mídias para a interface de adicionar abrir substituindo a administrativa ao invés de em um lightbox;
•Correção no serviço de Administração Automática onde a interface de Navegação não exibia a árvore correta em casos raros;
•Correção no serviço de Agenda de Sala de Reunião onde a interface de Agenda diária não era exibida adequadamente quando não havia sala cadastrada;
•Correção no serviço de Agenda de Sala de Reunião na interface de Agenda diária ao efetuar clique duplo em adicionar;
•Correção no controle de lum_orderedMultiItemList ao tentar adicionar um item excluído durante a mesma interação;
•Correção no uso do valor lum_currentUserName no DOUI;
•Correção no editor HTML onde a opção de inserir link para página, quando utilizada no modo de administração de conteúdo sem ter página de administração automática customizada, não exibia a árvore de canais e páginas do portal corretamente;
•Correção no cadastro de campo tipo file em conteúdo com versionamento;
•Correção no cadastro de campo tipo files que exibia mensagem de erro durante operações;
•Correção no acesso a operação REST em classe pertencente a módulo que foi atualizado alterando argumentos de uma operação da classe;
•Correção na inicialização do framework de REST para não impedir a inicialização do portal decorrente de classe REST inválida instalada em módulo;
•Correção nos exemplos de configuração do Apache Web Server contidos no manual no cenário em que a aplicação não está no contexto raiz;
•Correção no processamento de URLs de download de arquivos que tinham um comportamento inesperado ao utilizar cache de interface;
•Correção no serviço de Conteúdo Hierárquico que tinha um comportamento inesperado ao utilizar cache de interface;
•Correção no processo de sincronização de estrutura para evitar perda de definições de papéis de workflow do permissionamento de instâncias de serviço;
•Correção do processo de sincronização de estrutura para evitar erro ao atualizar transformação de arquivos;
