Estrutura de Diretórios e Arquivos gerados pelo Assistente

 Top  Previous  Next

Após a criação do serviço pelo assistente de criação, os seguintes arquivos são gerados:

•servicedefinition.xml:arquivo de definição do serviço e de suas interfaces;
•douidefinition.xml: arquivo que contém informações para manipulação de dados das interfaces;
•tables.sql: localizado no diretório “database”, contém um SQL apropriado para o respectivo banco de dados, e não é necessariamente compatível com o SQL ANSI;
•strings.pt_br.xml: localizado no diretório “strings”, contém a tradução do nome e descrição do serviço. As outras strings são traduzidas pelo framework. Caso deseja-se utilizar outros nomes para substituir o padrão, deve-se inserir tais strings neste arquivo.

Na figura a seguir, a estrurura “Customização/example_1” é o caminho definido no campo “Diretório de instalação” durante a criação do serviço pelo assistente. Ele é relativo ao “lumisdata/def”.
