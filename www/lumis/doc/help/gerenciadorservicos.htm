Gerenciador de Serviços

 Top  Previous  Next

Através do gerenciador é possível registrar os serviços desenvolvidos e atualizados para serem utilizados pelas soluções do portal. Os serviços do Lumis Portal já estão registrados e são atualizados a cada upgrade do produto.

Os serviços customizados somente estarão disponíveis para serem instanciados em um canal caso sejam registrados.
 

Observações

 

Os serviços que são criados pelo Assistente de Criação de Serviços de Gestão de Conteúdo são automaticamente registrados no portal;

 

A figura anterior exibe uma lista de serviços registrados, para os quais são exibidos o ID do serviço, Nome e Descrição. Todos estes estão definidos no servicedefinition.xml de cada serviço. Neste arquivo também encontram-se as definições de cada interface do mesmo.

Através do gerenciador de serviços é possível:
 

•Adicionar:

 

oCaminho do serviço: Para registrar um serviço basta apenas especificar o caminho relativo, após o diretório “lumisdata/def” onde está localizado o arquivo “servicedefinition.xml”.
 
•Deploy será apresentado em Deploy de Portlets;
 
•Atualizar: Quaisquer alterações no arquivo do serviços que implicam mudanças na funcionalidade do serviços, devem ser repassadas ao portal. Para isso, deve-se selecionar o arquivo e clicar em “Atualizar”;

 

Observação

 

A atualização do serviço gera um evento de expiração de cache em massa. Por padrão,as filas de cache utilizam uma configuração para que elas tratem esse tipo de expiração. Caso necessário, pode-se configurar as filas desejadas para que elas não sejam incluídas nas operações de expiração de cache em massa.

 

Ver mais em Filas de Cache.
 

 

 

•Excluir: Permite retirar o registro dos serviços não mais utilizados pelo portal. Desta forma, a instância do serviço não estará mais disponível na lista de instâncias de serviços.

Observações

 

1)Não é possível remover o registro de um serviço criado a partir do assistente de criação de serviços de gestão de conteúdo. A exclusão de tal serviço na administração do assistente de criação, remove automaticamente o seu registro no gerenciador de serviços;

2)A remoção do registro do serviço no Gerenciador de Serviços não implica a remoção dos seus arquivos físicos.
 

 

•Controle de Acesso: Permite configurar o controle de acesso do serviço selecionado, negar ou permitir o acesso a determinados grupos e usuários adicionados ao controle de acesso do serviço. Neste caso todas as instâncias de serviço terão as configurações de controle de acesso determinadas para o serviço, a não ser que em cada instância sejam atribuídas novas permissões para determinados grupos ou usuários.
