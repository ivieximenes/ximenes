Ao editar o canal:

•Criar Página(s) (Esta propriedade é visível somente na criação do canal): Permite adicionar uma ou mais páginas durante a criação do canal. Para isso, basta indicar os nomes das páginas, separados por “;”. As páginas criadas receberão os nomes indicados e sua posição no navegador será correspondente à sua ordem de cadastro. Por exemplo, especificando-se “Pag1;Pag2;Pag3” neste campo, serão criadas sob o novo canal três páginas: uma com nome “Pag1” e posição 0; outra com nome “Pag2” e posição 1; e por fim, “Pag3” e posição 2;
 
•Herdar Template: Permite ligar/desligar a herança do template de página padrão definida no canal “pai”;
 
•Template de Página Padrão: Quando a herança está ligada, é exibido o nome do template de página padrão herdado (não editável) que pode ser aplicado às páginas do canal. Quando a herança está desligada, permite atribuir a este canal um template de página padrão que pode ser aplicado em suas páginas ou, até mesmo, deixar indefinido. Porém existem dois cenários para atribuição de template de página padrão:

a) Atribuição do template de página padrão na criação do canal: o template será automaticamente aplicado às páginas do canal;

b)  Atribuição do template de página padrão na edição do canal: o template não será aplicado automaticamente a todas as páginas do canal. Para isso, em Aplicar Template de Página deve-se selecionar em quais páginas deseja-se ter o template aplicado.

 

Observação

 

Ao criar uma página, o campo "Aplicar Template" das propriedades da página vem preenchido automaticamente com o valor do template default do canal.

 

•Centralizar Páginas: Permite definir se todo o layout  das páginas será centralizado ou não, ou se essa propriedade será herdada do canal “pai”. O default do portal é não centralizar as páginas;

 

•Tipo de Layout das Páginas: Permite definir como o código HTML da página será gerado. As opções são:

 

1.Herdar Configuração: utiliza o valor definido para o canal pai.
2.Com tabelas: o código será gerado utilizando a tag html <table>.
3.Sem tabelas: o código será gerado utilizando a tag html <div> .
4.Arquivo de layout: o código será gerado a partir do arquivo de layout informado. O caminho do arquivo utilizado é relativo à raiz do classpath ou à pasta "lumisdata/shared/def". O arquivo de layout também pode estar em um módulo ou um tema. Para mais detalhes consulte o tópico Layout File. É sugerida uma lista de arquivos de layout que estejam na pasta www dos temas aplicados.

 

O valor default do framework  do portal é gerar código html com tabela.

 

É importante salientar que em páginas tableless o que é visualizado fora do modo administrativo do portal (fora do F2) pode ficar diferente do que é visualizado dentro do F2.
 

•Gerar Arquivo HTML: Permite determinar se as páginas pertencentes a tal canal, serão cacheadas apenas para usuário guest, todos usuários cadastrados, ou se irá herdar o que está definido no canal “pai” da página.

 

Nos casos de personalizações e entrada de dados, pode-se cachear a página apenas para usuário Guest . Dessa forma, para os usuários cadastrados, será exibida sempre a página dinâmica.

 

Observações

 
•A página a ser cacheada não deve conter interfaces com ordenação e filtros, por exemplo;
•O cache da página ficará desabilitado, não explicitamente, caso seja configurado para o canal, ou por herança, utilizar o protocolo HTTPS.
