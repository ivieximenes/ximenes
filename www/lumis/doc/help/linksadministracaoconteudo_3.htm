Administração de Conteúdo

 Top  Previous  Next

O serviço de Conteúdos Hierarquicos possui um ambiente administrativo disponibilizado pelo administrador do portal, para o cadastro do conteúdo pelo editor responsável. O conteúdo cadastrado pode ser alterado, visualizado ou excluído.

Informações básicas:

•Titulo: Título (obrigatório) linkável que compõe a lista de conteúdos.
•Introdução: Texto (opcional) que acompanha o título na interface de "Menu do Conteúdo".
•Imagem: Ilustração (opcional) que acompanha a introdução na interface de "Menu do Conteúdo".
•Conteúdo Pai: Permite informar qual será o conteúdo pai(já cadastrado) do item a ser cadastrado.
•Posição: Campo (obrigatório) que determina a posição do conteúdo entre seus irmãos na hierárquia. Caso um ou mais irmãos contenham o mesmo valor de posição, a ordem é determinada pelo título.
•Tipo do Link: Campo (obrigatório) que determina o comportamento do conteúdo na hierárquia. Os tipos são "Conteúdo Interno", "URL", "Página Interna", "Documento Interno" ou "Estrutural".

Tipos de conteúdo:

•Conteúdo Interno: Texto (obrigatório) interno do conteúdo que corresponde ao detalhe do mesmo. O usúario é remetido a este conteúdo ao clicar no título. O cadastro deste campo é realizado em um editor de texto que permite tipos de formatação semelhantes aos editores de texto conhecidos. Pode-se ainda editá-los em HTML através da opção "Codigo-Fonte" do editor de texto.
•URL: Link (obrigatório) a um site externo. O usuário é redirecionado ao clicar no título. O cadastro deste campo é realizado em uma caixa de texto que permite escrever a URL do site. Pode-se ainda escolher onde abrir o link: "Mesma janela", "Nova janela" ou "Pop-up". Quando Pop-up é selecionado, é possível adicionar propriedads ao pop-up.
•Página Interna: A página (obrigatório) da estrutura de canais para qual o usuário será remetido.
•Documento Interno: O documento (obrigatório) que este conteúdo representa. O link gerado para este tipo de conteúdo é o de download do documento.
•Estrutural: Contem apenas os campos comuns. Este tipo não gera link e serve apenas como um recipiente.
