Grupo

 Top  Previous  Next

Grupos no Lumis Portal são um mecanismo para facilitar o gerenciamento de direitos de acesso. Desta forma, usuários com o mesmo perfil de acesso devem ser agrupados, atribuindo permissões de acesso a determinado objeto ao grupo, ao invés de atribuir a cada usuário.

O Lumis possui os seguintes grupos padrão que por default já são cadastrados no portal:

•Administradores do Portal: agrupa os usuários com permissão de acesso de gerenciamento do portal (acesso ao portal studio, gestão de conteúdo, módulos e configurações). Caso um usuário ou outro grupo faça parte deste grupo, eles terão permissão de gerenciar todo o portal e suas configurações. Está contido em todos os outros grupos padrão do Lumis;
•Usuários cadastrados: grupo que contém automaticamente todos os usuários a virem a ser cadastrados no portal;
•Desenvolvedores: grupo que representa os desenvolvedores do portal, por exemplo;
•Todos os usuários: grupo que contém todos os usuários cadastrados mais usuário “guest”.

No gerenciador de grupos é possível visualizar todos os grupos cadastrados e executar as seguintes funções para administrar os cadastros:

 

•Adicionar: Permite incluir um novo grupo. Os seguintes campos podem ser preenchidos no formulário:

 

oApelido: Campo obrigatório que deve conter um nome curto para a identificação do grupo dinâmico. É considerado o shortID;
oNome: Campo obrigatório que deve conter o nome completo do grupo;
oDescrição: Breve descrição sobre o grupo. Não é obrigatório;
oTipo de Grupo: Campo obrigatório no qual deve ser escolhido o tipo de grupo para o grupo que está sendo cadastrado. Se para tal grupo serão adicionados membros manualmente, deve-se escolher o "Tipo de Grupo Padrão", que vem disponível com o produto. Para grupos cuja adição de membros será automática e dinâmica, deve-se escolher outro tipo de grupo que deve estar implementado e devidamente adicionado ao portal. A esses grupos dá-se o nome de Grupos Dinâmicos;
•Editar: Permite editar as propriedades de grupos globais e locais já cadastrados. Deve ser selecionado um de cada vez;
•Excluir: Permite excluir um grupo ou vários grupos selecionados na lista. Grupos locais são automaticamente excluídos do gerenciador de grupos, quando o canal em que eles foram criados, for excluído;
•Membro de: Permite visualizar os grupos aos quais o selecionado pertence, além de permitir adicioná-lo em outros, menos nos grupos "Todos os usuários" e "Usuários cadastrados". Caso necessário, é possível removê-lo dos grupos adicionados.

 

É possível visualizar também se tal grupo pertence a um outro indiretamente, por transitividade. Por exemplo, o grupo "Documentadores" pertence ao grupo "Publicadores" (grupo direto de "Documentadores") e, este pertence ao grupo "Jornalistas". Por transitividade, o grupo "Documentadores" pertence ao grupo "Jornalistas" (grupo indireto de "Documentadores").

 

 

•Membros: Exibe os usuários ou grupos membros do grupo selecionado. Além disso, é possível adicionar usuários e grupos já cadastrados no portal (globais ou locais aos quais o administrador tem acesso), para fazerem parte do grupo selecionado. Outra opção é a de exclusão de usuários e grupos do grupo selecionado. Porém não serão removidos do portal, apenas deixam de fazer parte de tal grupo.

Através do filtro “Domínio” é possível verificar quais usuários ou grupos são globais e quais são locais de acordo com a permissão de acesso a estes pelo administrador.

 

Observação

 

Não é possível adicionar manualmente membros a grupos dinâmicos.

 

•Controle de Acesso: É possível agregar grupos e usuários ao controle de acesso do grupo selecionado e para cada um deles atribuir permissões para o seu gerenciamento. Se a opção de herança estiver habilitada, estará herdando o controle de acesso global do portal. Neste caso será possível negar as permissões herdadas e adicionar novos usuários ou grupos no controle de acesso, atribuindo-lhes permissões.
