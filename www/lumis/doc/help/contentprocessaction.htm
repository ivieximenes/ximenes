Como verificado em DOUI-->Process Actions, a execução dos mesmos é realizada pelo ProcessActionContainer que é responsável por instanciar as classes que implementam a interface IProcessActionHandler .

Porém os process actions handlers padrão de doui para adicionar, atualizar e excluir dados não podem ser usados em sources de content, porque estes precisam também manipular dados específicos de metadados. No lugar de cada um desses process actions handlers de DOUI há um de content que deve ser usado em seu lugar quando o source for do tipo “contentTable”.

A tabela abaixo exibe as classes de process actions handlers que devem ser utilizadas no lugar de cada uma de DOUI.
