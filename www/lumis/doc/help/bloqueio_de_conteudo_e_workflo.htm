Bloqueio de Conteúdo e Workflow

 Top  Previous  Next

Bloqueio de um conteúdo pode afetar transições entre estados de um workflow.  Como comportamento padrão para as transições de estados de um workflow, se o conteúdo estiver bloqueado para um publicador ao ser enviado para o próximo estado, o conteúdo será automaticamente desbloqueado, independente do tipo de bloqueio.  Este comportamento, porém, é configurável nos arquivos de XML de workflow (workflowdefinition.xml, caso seja usado o workflow padrão do lumis).  Para cada transição que não vai seguir o comportamento padrão, deverá ser colocado no elemento <transition /> o atributo lockAction, cujos possíveis valores são: unlock (sempre realizar o desbloqueio na transição) e keepCurrent (manterá o estado do bloqueio, caso no próximo estado do workflow, o usuário tenha acesso ao conteúdo, caso contrário, o conteúdo será desbloqueado).  Se o atributo lockAction não for explicitado, o valor unlock é utilizado implicitamente.
