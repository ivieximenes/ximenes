Contextos

 Top  Previous  Next

O contexto de negócio é uma estrutura customizável que poderá disponibilizar para os serviços do Lumis informações referentes à sessão do usuário.

Um contexto pode conter, por exemplo, uma lógica de negócio capaz de informar o ‘tipo’ de device sendo usado pelo usuário.

 

 

Contextos precisam ser registrados antes de ser utilizados. Este processo é similar ao registro de serviços: o usuário informa o ‘path’ do arquivo de definição do contexto para registrar ou alterar o registro.

 

•Adicionar

 

                             

Caminho do contexto: Para registrar um contexto basta apenas especificar o caminho relativo, após o diretório “lumisdata/def” onde está localizado o arquivo “contextdefinition.xml”. Para saber como criar um contexto de negócio veja em Contexto de Negócio em Customização e Desenvolvimento.

 

Observações

 

Registrar um contexto não o aplica a nenhum website no Lumis Portal – cabe ao montador ou administrador fazer esta correlação, informando que o contexto será aplicado a um ou mais websites.

 

•Validar Páginas Quebradas

Ao selecionar um contexto e clicar nesta funcionalidade, todas as páginas que utilizarem o contexto serão validadas, verificando se há alguma interface cujo cache exija um atributo inexistente no contexto ou que não possa ser calculado por ele (classe Java utilizada inexistente ou erro de sintaxe nas ELs, por exemplo).

 

 

•Atualizar

Quaisquer alterações no arquivo do contexto devem ser repassadas ao portal. Para isso, deve-se selecionar o arquivo e clicar em “Atualizar”;

 

•Excluir

Permite retirar o registro dos contextos não mais utilizados pelo portal. Desta forma, o contexto não estará mais disponível para ser utilizado pelos serviços.
