Deployment

 Top  Previous  Next

O Lumis Portal provê ferramentas que visam facilitar o deployment de projetos em um dado ambiente. O portal utiliza-se de três níveis de artefatos para fazer o deploy de projetos: componentes, módulos e pacotes. Cada um desses é descrito abaixo.

 

 

Componentes

Componentes são unidades de aplicações que podem possuir uma versão para controle, declaração de dependência de outros componentes, pacotes Java, serviços, temas, contextos de negócio, estrutura de canais e arquivos públicos (arquivos que ficarão nos diretórios estáticos dos websites). Não existe deployment de componentes puros no portal. Um componente deve sempre pertencer a um módulo e para que o componente seja instalado no portal, seu módulo deve ser instalado. Nesse caso, a instalação do módulo acarretará na instalação do componente também.

Um componente é definido pelo seu arquivo de definição. Para maiores informação sobre componentes, veja a seção Componentes.

O portal provê uma lista de todos os componentes instalados no portal. Para informações da lista de componentes, veja a seção Gerenciador de componentes.

 

 

Módulos

Módulos são unidades de deployment de aplicações que podem conter pacotes Java, serviços, temas, contextos de negócio e componentes. Ao instalar (ou atualizar) um módulo, todos os seus artefatos serão disponibilizados. Suas classes Java ficarão disponíveis em um classloader, seus serviços, seus temas, seus contextos de negócio e seus componentes serão registrados e disponibilizados. Um módulo deve conter um arquivo de definição que traz informações importantes sobre ele. Para maiores informações sobre módulos, veja a seção Módulos.

O portal provê uma lista administrativa de todos os módulos do portal. Para informações sobre a lista de módulos, veja a seção Gerenciador de módulos.

 

 

Pacotes

Pacotes são unidades de deployment de aplicações que podem conter scripts de instalação, módulos e comandos para ajustes diversos. Para maiores informações sobre pacotes, veja a seção Pacotes.

O portal provê um histórico de instalações de pacotes no portal. Para informações sobre o histórico de pacotes, veja a seção Histórico de pacotes.

 

 

Recomendações

É altamente recomendado que a seção de Recomendações seja lida, pois a mesma contém informações úteis e importantes.

 

Observação
